1. Disección de la Fase 1: Los 6 Pilares Operativos
Este proyecto busca transformar la gestión de anuncios de un proceso manual y subjetivo a una arquitectura autónoma basada en datos.
Pilar
	Descripción Técnica
	Impacto Esperado
 
	Despliegue Masivo
	Traducción de filas de Google Sheets a solicitudes HTTP POST hacia la Meta Graph API.
	Reducción de carga de 45 min a 5 segundos.
	Ingeniería de Imagen
	Carga automatizada de archivos multimedia para la obtención de image_hashes.
	Eliminación de errores manuales en la carga de creatividades.
	Espionaje Algorítmico
	Scraping de la Facebook Ad Library para detectar tendencias en el sector fashion.
	Base de datos de mercado actualizada sin navegación manual.
	Síntesis de Creativos
	Uso de Gemini 2.5 Flash para generar 5 variaciones de copy basadas en datos ganadores.
	Ingeniería inversa de ganchos (hooks) exitosos.
	Extracción de Métricas
	Consultas cíclicas vía API (cada 3 horas) para extraer Spend, ROAS y CPA.
	Visibilidad del rendimiento en tiempo real sin reportes CSV.
	Filtro de Alertas
	Lógica condicional que dispara webhooks a Slack ante CPA crítico.
	Gestión por excepción: solo se actúa cuando algo falla.
	2. Roadmap Técnico: Secuencia de Implementación
El orden de implementación prioriza la estabilidad de la infraestructura antes de añadir la capa de inteligencia.
1. Capa de Datos (Foundation): Configuración del Dashboard en Google Sheets como "Single Source of Truth".
2. Capa de Orquestación: Despliegue de n8n (preferiblemente auto-hospedado) para el manejo de flujos.
3. Integración de Comunicación: Configuración de la Meta App, generación de Tokens de Acceso de Larga Duración (User Access Token) y permisos de Ads Management.
4. Capa de Inteligencia: Integración de la API de Google Gemini para el procesamiento de lenguaje natural y limpieza de JSON.
5. Capa de Notificación: Configuración de Webhooks de Slack para el reporte operativo.
3. Guía de Implementación (Boris Tane Framework)
Esta sección sirve como guía paso a paso para el empleado a cargo del desarrollo.
3.1. Research (Investigación)
* Estudiar la documentación de la Meta Graph API v23, específicamente los endpoints de /campaigns, /adsets y /ads.
* Revisar los nodos de Google Sheets y HTTP Request en n8n.
* Familiarizarse con el formato de respuesta JSON de Meta para identificar los campos id y status.
3.2. Plan (Planificación)
* Dibujar el flujo lógico: ¿Qué pasa cuando una fila en Sheets cambia a estado "Pendiente"?
* Mapear la jerarquía de creación: Campaña -> Adset -> Creative -> Ad. Cada ID obtenido en un paso debe ser inyectado en el siguiente.
* Definir los umbrales de alerta: ¿A partir de qué CPA (Costo por Adquisición) debe Slack enviar una notificación roja?
3.3. Annotate (Anotación y Estructura)
Antes de programar, definir los objetos JSON base. Ejemplo de estructura para la creación de un anuncio:
{
 "name": "Anuncio_BH_001",
 "creative": {"ad_creative_id": "ID_DEL_CREATIVE"},
 "adset_id": "ID_DEL_ADSET",
 "status": "PAUSED"
}

Documentar las funciones de JavaScript necesarias dentro de los nodos "Code" de n8n para limpiar caracteres especiales en los copies generados por la IA.
3.4. Implement (Implementación)
* Paso 1: Construir el nodo que lee las filas de Sheets con el filtro Estado = 'Pendiente'.
* Paso 2: Implementar el nodo de carga de imagen para obtener el hash.
* Paso 3: Crear la secuencia de nodos HTTP Request para la jerarquía de Meta.
* Paso 4: Configurar el nodo de Gemini para recibir las métricas crudas y devolver un resumen ejecutivo.
* Paso 5: Realizar pruebas en una cuenta de Sandbox antes de pasar a la cuenta oficial de BH FASHION.
Este documento es una guía viva. Cualquier desviación técnica detectada durante la fase de Implementación debe ser anotada en la sección de Research para iterar el modelo.
